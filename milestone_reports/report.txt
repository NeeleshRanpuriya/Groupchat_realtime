============================================================
1. ABSTRACT
============================================================
This project develops an AI-powered real-time chat moderation platform that detects toxic content, classifies message intent, analyzes communication tone, and suggests polite rewrites. The system is built using FastAPI, WebSockets, SQLAlchemy, and Next.js with TypeScript. It is designed to improve online conversation quality by combining automatic moderation with user coaching in a live chat environment.

============================================================
2. PROBLEM FORMALIZATION
============================================================
2.1 Problem Statement
Modern chat platforms face challenges such as abusive language, aggressive communication, and unclear intent. Manual moderation does not scale for active communities and cannot provide immediate feedback.

2.2 Formalized Problem
Given a message m from user u in room r at time t, the system must:
- Detect whether the message is toxic.
- Classify user intent.
- Identify communication tone.
- Generate coaching and polite rewrite suggestions when needed.
- Deliver low-latency moderation feedback during real-time communication.

2.3 Objectives
- Build a real-time AI moderation pipeline.
- Improve conversation quality through constructive guidance.
- Store moderation history for tracking and analytics.
- Provide a practical full-stack implementation suitable for academic and real-world use.

2.4 Scope
In scope:
- Toxicity detection
- Intent classification
- Tone analysis
- Coaching and rewrite suggestions
- WebSocket-based real-time chat
- User authentication and room-based communication
- Message history and moderation statistics

Out of scope:
- Large-scale distributed architecture
- Advanced multilingual production moderation
- Human-in-the-loop moderation workflow integration

2.5 Constraints
- External API dependency for advanced tone analysis (OpenAI).
- Latency and performance trade-offs for NLP inference.
- Limited dataset size for custom intent and rewrite tasks.

============================================================
3. COMPLETE DESIGN
============================================================
3.1 System Architecture
The system follows a full-stack client-server architecture:
- Frontend: Next.js + TypeScript + Tailwind CSS
- Backend: FastAPI + WebSockets + SQLAlchemy
- Database: SQLite/PostgreSQL-compatible ORM model
- AI Modules: Toxicity detector, intent classifier, tone analyzer, rewrite generator

3.2 Workflow Design
1. User sends message through frontend WebSocket client.
2. Backend receives message and executes moderation pipeline.
3. Toxicity score, intent type, and tone are inferred.
4. Coaching and polite rewrite are generated when required.
5. Message and analysis metadata are saved in database.
6. Final enriched message is broadcast to room participants in real time.

3.3 Backend Module Design
- main.py: API routes, authentication, room/chat flow, moderation integration
- models.py: User, ChatMessage, MessageReaction entities
- database.py: connection/session and DB initialization
- toxicity_detector.py: transformer + rule-based Hinglish abuse detection
- intent_classifier.py: pattern and keyword-driven intent model
- tone_analyzer.py: OpenAI analysis with fallback-based coaching/rewrite

3.4 Frontend Module Design
- app/page.tsx: landing flow and room entry UX
- app/chat/[room_id]/page.tsx: chat room state, WebSocket messaging, moderation panels
- components: message rendering, input actions, analytics panel, user list, group tools

3.5 Data Model Summary
User:
- id, username, email, password hash, bio, active status, created_at

ChatMessage:
- id, username, message, room_id, timestamp
- toxicity_score, is_toxic
- intent, intent_confidence
- tone, tone_confidence
- coaching_message, suggested_rewrite
- edit/pin/reply/file metadata

MessageReaction:
- id, message_id, username, reaction, timestamp
